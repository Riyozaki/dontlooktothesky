# Список ресурсов для проекта

**Статус:** рабочий список v0.1

## 1. Что желательно подготовить автору

### Логотип

1. Основной логотип `dontlooktothesky_logo.png`
   - прозрачный фон;
   - ширина 1600–2400 px;
   - светлая версия для тёмного меню;
   - без встроенного фона.

2. Компактная версия `dontlooktothesky_mark.png`
   - квадрат или почти квадрат;
   - для небольших кнопок, загрузочного экрана и возможного окна игры.

3. При необходимости тёмная версия логотипа
   - только если основная светлая плохо читается на светлых фонах.

### Иконки интерфейса

Для каждой иконки желательно иметь состояния idle, hover и pressed. Если отдельных состояний не будет, hover сделаем программным.

Обязательные:

- назад;
- вперёд;
- история;
- автопереход;
- пропуск;
- сохранение;
- загрузка;
- настройки;
- главное меню;
- закрыть;
- подтверждение;
- отмена.

Желательные:

- избранная реплика или закладка;
- индикатор новой строки в истории;
- значок звука;
- значок музыки;
- значок полноэкранного режима.

Формат: предпочтительно SVG или PNG с прозрачностью. Для PNG достаточно 128–256 px по большей стороне.

### Шрифт

Не обязательный ресурс: до выбора авторского шрифта используем временный системный.

Если будет выбран отдельный шрифт, нужны:

- файл `.ttf` или `.otf`;
- подтверждение лицензии;
- поддержка кириллицы;
- обычное начертание;
- при возможности полужирное начертание.

## 2. Что я подготовлю сам

- программные трансформы активного и неактивного говорящего;
- затемнение слушающего персонажа;
- диалоговую панель;
- именные плашки;
- кнопки выбора;
- базовые UI-плашки без внешних изображений;
- переходы и затемнения;
- Ren’Py-реализацию истории, сохранений и настроек;
- технические заглушки для отсутствующих SFX;
- манифест лицензий и источников.

- вибрация телефона;
- уведомление;
- застёгивание рюкзака;
- подъездная дверь;
- велосипедный замок;

- падение штукатурки;
- движение мусорной машины;
- сканер депо;
- движение коробки и ленты.

### C01

- банковский терминал;
- подпись на экране;
- домофон;
- звонок телефона;
- велосипед по мокрому асфальту;
- открытие аптечного пакета;
- дождь;

### C02

- дождь по капюшону;
- автомобиль на мокрой дороге;

- шум остановки;

SFX можно добавлять после первичного UI-прохода. Отсутствие этих файлов пока не блокирует разработку.

## 4. Источник SFX

Первый кандидат для отбора — архив Sonniss #GameAudioGDC. Перед использованием каждого набора сохраняем:

- официальный URL;
- название набора;
- дату загрузки;
- ссылку на лицензию;
- список выбранных файлов;
- контрольную сумму архива или файлов, если это удобно.

В игру копируются только нужные обработанные эффекты. Исходные архивы не должны попадать в игровую сборку.

Нельзя распространять выбранные SFX как отдельную библиотеку вне готовой игры.
