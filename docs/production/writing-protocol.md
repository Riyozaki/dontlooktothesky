# Протокол написания

## Перед художественным текстом

1. Прочитать минимальный набор, описанный в `../state/project-state.md`:
   - `../bible/voice-and-characters.md`
   - `../bible/character-bible.md`
   - `../filters/recursive-check.md`
   - `../production/production-method.md` (этот документ)
   - `../story/story-structure.md`
   - `../story/story-map.md`
   - `../story/routes/` — концепция активного маршрута
2. Проверить, что задача соответствует `../state/project-state.md`.
3. Обсудить с автором новые сюжетные узлы, если они влияют на финал, отношения, моральную цену или физическую механику.
4. Составить карту блока без литературных формулировок:
   ```text
   сцена
   цель героя
   препятствие
   действие
   скрытый план
   физическое изменение
   последствие
   ```
5. Отдельно выписать голоса участников сцены.

## Тест до главы

Не писать сразу 4 000–5 000 слов.

Сначала подготовить:

- один короткий сюжетный синопсис блока;
- 3–5 ключевых конфликтных точек;
- один тестовый фрагмент на 700–1 000 слов;
- голосовой тест реплик персонажей;
- список потенциальных физических рисков.

После читательского одобрения теста писать полный блок.

## Во время написания

- Художественный источник — только `game/*.rpy`.
- Не вставлять производственные метки в прозу.
- Не писать ради плановой цифры.
- Не превращать персонажа в носитель нужной экспозиции.
- Не считать любую бытовую сцену романтикой.
- Не создавать опасность только для смены музыки.
- Каждая манипуляция должна оставить след.
- Каждая помощь должна иметь цену или изменить доверие.
- Если персонаж говорит «я не знаю», проверить, что это его голос, а не удобный способ скрыть недописанную информацию.

## После тестового фрагмента

Проверить в таком порядке:

1. читательская реакция без плана;
2. физика и пространство;
3. авторский голос;
4. отдельные голоса персонажей;
5. анти-ИИ-фильтр;
6. причинность и рекурсия;
7. только потом Ren’Py и Markdown.

## Границы утверждения

- Черновой текст не считать каноном.
- Рабочий блок не считать готовым из-за lint.
- Не переходить к следующей главе, если первая не прошла голосовой и читательский проход.
- Не переписывать сцену вручную в Markdown: зеркало генерируется автоматически.
- После крупного блока делать производственный отчёт.

## Фиксация

После одобрения:

```text
1. generate mirror
2. run transfer audit
3. run static validation
4. run Ren’Py lint/compile
5. update active state
6. commit to arena branch
7. push only to arena branch
```

Merge в `main` не выполняется.
