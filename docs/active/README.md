# ACTIVE — рабочий пакет проекта

Эта папка содержит единственный обязательный пакет контекста для новой художественной работы.

## Читать каждый раз перед написанием

1. `00-project-state.md` — что за проект и на каком мы этапе.
2. `01-voice-and-characters.md` — голос автора, портреты и речевые профили.
3. `02-filters-and-recursive-check.md` — фильтры и порядок рекурсивного чтения.
4. `03-story-map.md` — макроструктура и уже зафиксированный канон.
5. `04-writing-protocol.md` — производственный протокол и стоп-условия.
6. `05-valeria-route-brief.md` — активная концепция маршрута Валерии.

## Порядок источников

```text
01. Последние прямые решения автора.
02. Этот ACTIVE-пакет.
03. Утверждённый текст в game/*.rpy.
04. Полные справочные документы.
05. Старые планы и исторические аудиты.
```

Если старый план противоречит этому пакету или последнему решению автора, старый план не используется.

## Что не читать автоматически

- весь `docs/manuscript/`;
- `docs/reference/`;
- старые аудиты и планы;
- все главы маршрута целиком до появления конкретной задачи.

Эти материалы открываются адресно: для сверки факта, непрерывности или отдельного рекурсивного прохода.

## Техническая истина

Художественный текст пишется один раз в `game/*.rpy`. Markdown в `docs/manuscript/` — автоматически сгенерированное зеркало через `scripts/extract_rpy_manuscript.py`.
