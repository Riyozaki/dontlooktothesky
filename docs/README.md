# docs/

Организационная и сюжетная документация проекта.

## Структура

| Папка | Что внутри | Когда читать |
|---|---|---|
| [`bible/`](bible/) | Голос автора, речевые профили, библия персонажей | Перед любой художественной работой |
| [`filters/`](filters/) | Фильтры клише и рекурсивные проходы R0–R6 | При финальной вычитке сцен |
| [`production/`](production/) | Методика и протокол производства | Перед началом главы, при стыковке блоков |
| [`state/`](state/) | Текущее состояние проекта | В начале сессии, чтобы понять этап |
| [`story/`](story/) | Макроструктура, посценовые планы, маршруты, beat-sheets | При планировании сцен |
| [`manuscript/`](manuscript/) | Автогенерируемые зеркала .rpy-файлов | По конкретной задаче, не автоматически |
| [`reference/`](reference/) | Указатель на внешние справочные материалы | По запросу |
| [`licenses/`](licenses/) | Лицензии используемых ассетов | При юридических вопросах |
| [`archive/`](archive/) | Устаревшие и дублирующие документы | Только если нужна историческая справка |

## Что считать источником правды

1. Последние прямые решения автора.
2. Корневые `SKILL.md`, `anti-cliche-filter.md`, `examples.md`.
3. Утверждённые документы в `bible/`, `filters/`, `production/`, `state/`, `story/`.
4. Художественный текст в `game/*.rpy`.
5. Зеркала в `manuscript/` — для чтения, но **не для правки** (правки вносятся в `.rpy`).

`manuscript/*.md` обновляется через `scripts/extract_rpy_manuscript.py` и не редактируется вручную. См. [`manuscript/README.md`](manuscript/README.md).
