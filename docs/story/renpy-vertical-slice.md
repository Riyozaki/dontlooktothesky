# Ren’Py — рабочий вертикальный срез

**Статус:** текстовый вертикальный срез C00–C03 интегрирован; визуальный и runtime-проход впереди

## Что реализовано

- минимальная конфигурация проекта 1920×1080;
- временные экраны главного меню, диалога, выборов, истории, сохранения и настроек;
- определения основных персонажей;
- регистрация фонов, спрайтов и музыкальных тем;
- рабочие трансформы для исходников разного разрешения;
- полноценные сценарные файлы `c00.rpy`, `c01.rpy`, `c02.rpy`, `c03.rpy`;
- перенос канонической рукописи C00–C03 в Ren’Py;
- показ Александра во время его реплик;
- выборы и переменные общей ветки;
- музыка и базовые переходы между сценами;
- участие спрайтов Александра, Мираэль, Лены, Артёма и Дмитрия.

Новая литературная версия хранится в `docs/manuscript/c00.md`–`c03.md`. Файлы `.rpy` являются её игровой реализацией, а не отдельным источником текста.

## Структура

```text
game/
├── definitions.rpy  # персонажи, ресурсы и состояние
├── gui.rpy          # временные размеры и цвета UI
├── options.rpy      # конфигурация проекта
├── screens.rpy      # минимальные экраны
├── script.rpy       # точка входа
├── c00.rpy          # утро, депо и накладная
├── c01.rpy          # рабочий маршрут и последний заказ
├── c02.rpy          # авария и мёртвый интервал
└── c03.rpy          # квартира, Лена и первая ночь
```

## Ресурсы во время разработки

Исходные изображения и музыка пока лежат в корне репозитория. В `.rpy` используются относительные пути вида `../images/...` и `../Track.mp3`.

Это позволяет не дублировать около 250 МБ бинарных файлов. Перед релизной сборкой потребуется:

1. создать оптимизированный каталог `game/assets`;
2. скопировать туда игровые версии изображений и звука;
3. заменить временные относительные пути;
4. оставить исходные мастера отдельно;
5. проверить сборку Windows на чистой машине.

## Как запустить

1. Установить Ren’Py SDK актуальной стабильной версии.
2. Добавить каталог репозитория как существующий проект либо разместить его в каталоге проектов Ren’Py.
3. Запустить проект `dontlooktothesky` из Launcher.
4. При первом запуске проверить доступ к ресурсам, кириллицу, историю и сохранения.

В рабочем окружении Arena Ren’Py SDK отсутствует, поэтому интерактивный запуск здесь пока не выполнялся. Доступные проверки запускаются командой `python3 scripts/validate_renpy_static.py`. Скрипт проверяет метки, вызовы, кавычки, случайные закрывающие скобки и пути к ресурсам.

## Что проверять вручную

- фон заполняет экран без нежелательной деформации;
- спрайты имеют единую нижнюю привязку;
- текст не выходит за диалоговое окно;
- кнопки выбора помещают длинные русские формулировки;
- музыка плавно меняется между сценами;
- откат корректно возвращает значения флагов;
- сохранение и загрузка работают после каждого выбора;
- постоянные переменные переживают новую игру;
- авария читается без полноценного CG;
- временный полный трек Наблюдателя в C02 позже заменяется коротким фрагментом.
